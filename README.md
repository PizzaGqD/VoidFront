# VoidFront

Браузерная IO-стратегия: одиночка с ботами и мультиплеер (лобби до 4 игроков, форматы 1v1 и 1v1v1v1).

## Как запустить

Проект нужно открывать **по HTTP**, а не через `file://`.

### Мультиплеер (рекомендуется)

Сервер раздаёт игру и лобби на одном порту:

```bash
cd "c:\Users\7StepToHell\Desktop\DTF\public\Hoi2D"
npm install
npm start
```

Открой в браузере: **http://localhost:3040**

- Меню → Мультиплеер → введи ник → «Создать комнату» или выбери комнату из списка «Открытые комнаты» / введи код и «Войти».

### Как сделать так, чтобы лобби увидели другие

Все должны заходить в игру **с одного и того же сервера**.

1. **Один компьютер**  
   Все открывают **http://localhost:3040** — список «Открытые комнаты» общий, можно заходить в созданные комнаты.

2. **Несколько компьютеров в одной сети (Wi‑Fi или Radmin VPN)**  
   - Запусти сервер (`npm start`), создай комнату. В лобби в блоке «Адрес для подключения» будет твой IP и кнопка «Копировать».  
   - Если используешь **Radmin VPN** (или Hamachi и т.п.) — сервер сам подставит твой VPN-адрес (26.x.x.x для Radmin), чтобы второй игрок заходил по нему.  
   - Другой игрок открывает в браузере эту ссылку (например **http://26.x.x.x:3040** или **http://192.168.x.x:3040**), вводит ник и заходит в комнату по коду или из списка.

3. **Игра открыта с другого сайта/порта**  
   Если страница загружена не с `http://IP:3040`, в консоли браузера (F12) задай перед загрузкой:
   ```js
   window.GAME_SERVER_URL = "http://IP_СЕРВЕРА:3040";
   ```
   затем обнови страницу. Тогда клиент будет подключаться к лобби на этом сервере.

### Два роутера: другой комп не видит хост

Если у вас **два роутера** (например: интернет → основной роутер → кабель к компу супруги, и отдельно второй роутер → твой комп), компьютеры оказываются в разных подсетях. С одного ПК другой может быть «невидим».

**Что сделать:** сервер должен работать на том компе, который **напрямую подключён к основному роутеру** (или к тому роутеру, к которому подключены оба).

- **Вариант А:** Запустите `npm start` и создайте комнату **на компе супруги** (он на основном роутере). На своём компе откройте в браузере **http://IP_СУПРУГИ:3040** (IP супруги узнайте на её ПК: `ipconfig` → IPv4). Подключение с твоего ПК к её IP обычно проходит.
- **Вариант Б:** Подключите свой ПК к **основному** роутеру (кабелем или Wi‑Fi), чтобы оба были в одной сети — тогда подойдёт любой хост.
- **Вариант В:** Включите на втором роутере режим **точки доступа (AP)** и отключите у него свой DHCP — тогда твой ПК получит IP от основного роутера и снова будет в одной сети с супругой.

### Игра по интернету (туннель)

Чтобы играть **не по локальной сети**, а по интернету (разные города, разные провайдеры), запусти сервер с туннелем:

```bash
npm install
npm run tunnel
```

В консоли появится ссылка вида **https://xxxx.loca.lt** — это публичный адрес твоего сервера. Отправь её тому, с кем играешь (в мессенджер, почту). Второй игрок открывает эту ссылку в браузере, вводит ник, видит твою комнату в списке или входит по коду. Роутер настраивать не нужно.

- При первом открытии ссылки loca.lt может показать страницу «Click to continue» — нажать кнопку, после чего откроется игра.
- Туннель работает, пока запущен `npm run tunnel`; ссылка меняется при каждом новом запуске.
- Обычный запуск без туннеля: `npm start` (только локальная сеть).

### Только одиночка (без мультиплеера)

Можно раздавать папку любым HTTP-сервером (без Node-сервера лобби):

```bash
npx --yes serve
# или: python -m http.server 8080
```

Открой указанный адрес (например http://localhost:3000). В меню будет доступна только «Одиночка с ботами».

## Runtime config и релиз

Для production-сборки есть команда:

```bash
npm run build:release
```

Она собирает статический клиент в `dist/client` и исключает:

- `tests/`
- `VFX_LAB/`
- `server.js`
- `.edge-headless-profile*`
- прочий локальный dev-мусор

В release bundle файл `runtime-config.js` уже переключает билд в production-safe режим:

- debug/test UI выключены
- perf HUD выключен
- `window._gameTest` не публикуется

### Продовый backend config

Если клиент раздаётся через `server.js`, можно задавать runtime config через переменные окружения:

```bash
set GAME_SERVER_URL=https://your-backend.example
set VOIDFRONT_AUTHORITY_MODE=host-client
set RELEASE_CHANNEL=production
set VOIDFRONT_ENABLE_DEBUG_TOOLS=false
set VOIDFRONT_ENABLE_TEST_UI=false
set VOIDFRONT_ENABLE_PERF_HUD=false
npm start
```

Сервер сам отдаст `runtime-config.js` с этими значениями.
